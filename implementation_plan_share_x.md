# План исправлений: score PNG и публикация в X

## Цели
1. Корректно отрисовывать `best score` на `img/score_result.png` в нужной зоне и стиле (SVG-числа, градиент, шрифт, размер).
2. Публиковать пост в X с реальным прикреплённым изображением через OAuth user context.

## Шаги
1. Уточнить и зафиксировать layout-параметры (anchor, baseline, box) в `utils/shareCard.js`.
2. Переписать SVG-оверлей:
   - добавить `linearGradient` для цифр;
   - убрать агрессивное `textLength`-сжатие;
   - позиционировать по центру target box;
   - динамически подбирать размер шрифта в допустимом диапазоне.
3. Усилить X media flow:
   - если media upload не вернул `media_id_string`, не публиковать текстовый твит, а вернуть ошибку;
   - добавить безопасные диагностические логи по upload/create tweet.
4. Обновить/добавить тесты для share-result на кейс отсутствующего media_id.
5. Сгенерировать тестовый PNG локально и в следующей итерации отдать прямую ссылку/путь для проверки вне приложения.

## Риски
- Визуальные расхождения из-за отсутствия точного font file в окружении.
- Различия в поддержке upload endpoint в X API при разных app-настройках.

## Критерии готовности
- Цифры крупные, в нужной позиции, с градиентом.
- `POST /api/x/share-result` не публикует твит без media и явно сообщает о проблеме.
